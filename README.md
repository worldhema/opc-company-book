# AI 原生公司 · 全职能 AI 使用手册

> 目标：让整个组织**原生使用 AI、运行在 AI 之上**。人负责判断、决策与把关，AI 负责跑量、提速与初筛。
> 面向中国市场，工具推荐口径为「海外标杆为主 + 国内平替」，读者定位为**一线执行者**（拿来即用）。

## 全书结构

整条流水线：**需求 → 产品 → 设计 → 研发 → 发布 → 媒体推广**

每一章统一采用五模块结构：

1. **角色设定**：岗位定位、职责边界、人做什么 vs AI 做什么
2. **Skills**：该角色必须具备的 AI 协作能力
3. **AI 工具使用说明**：最多 3 个（海外标杆 + 国内平替）
4. **工作流程**：可直接执行的 SOP（带提示词模板）
5. **交付物**：本环节产出 → 交给下一环节的输入

## 交付物咬合关系（流水线 + 横向统筹层）

**纵向：单个产品从需求到推广的流水线（第一至六章）**

```
需求清单 v1  ──►  产品文档(PRD)  ──►  设计稿+规范  ──►  可运行代码  ──►  发布上线  ──►  传播扩散
  (需求岗)         (产品岗)          (设计岗)         (研发岗)        (发布岗)      (推广岗)
```

**横向：多产品并行时的统筹层（第七章）**

项目管理不是流水线里的一环，而是**贯穿全流程、协调多条流水线**的横向层。当公司有多个产品同时跑时，它负责资源调度、进度对齐、风险管控、跨产品协同：

```
┌─────────────────────────────────────────────────────────┐
│              第七章 · 项目管理（横向统筹层）                  │
│   资源调度 · 进度对齐 · 风险预警 · 跨团队协同 · 组合优先级      │
└─────────────────────────────────────────────────────────┘
        ↑          ↑          ↑          ↑          ↑
    产品A流水线  产品B流水线  产品C流水线   ...    （每条都是上面的六步链路）
```

**底座：支撑所有职能的通用基建（第八章）**

工具选型、数据安全、提示词沉淀、成本管理，是所有角色共用的地基。没有它，各团队各自为政，埋下合规和成本隐患：

```
┌─────────────────────────────────────────────────────────┐
│   纵向流水线（1-6章）  +  横向统筹层（7章）                    │
├─────────────────────────────────────────────────────────┤
│              第八章 · 通用基建（底座层）                      │
│   工具选型原则 · 数据安全与合规 · 提示词库沉淀 · 成本管理        │
└─────────────────────────────────────────────────────────┘
```
