# 第三章：设计（Design）

> 承接产品岗的 PRD，设计岗要把"文字描述的功能"变成"可看、可点、可交付给研发"的设计稿。AI 原生模式下，设计师不再是"从零画每一个像素的人"，而是"用 AI 快速生成信息架构、交互原型和视觉方案，自己专注在体验判断、审美把关和一致性维护"的人。

---

## 0. 本章要解决的核心问题

**如何把 PRD 变成一套设计？**

PRD 交来的是文字：功能详述、页面说明、字段规则、业务流程。设计岗要把它转化成四层递进的产物：

1. **信息架构（IA）**：产品有哪些页面、页面之间怎么跳转、内容怎么组织
2. **交互流程（UX Flow）**：用户在每个页面上怎么操作、点了之后发生什么
3. **视觉设计（UI）**：每个页面长什么样——布局、颜色、字体、组件、图标
4. **设计规范 + 标注**：颜色/字号/间距/组件规范 + 研发可用的切图和标注

AI 的作用，是把"文字 → 结构 → 原型 → 高保真"这条链路的重复劳动接管，让设计师把精力放在**体验是否顺畅**和**视觉是否统一**上。

---

## 1. 角色设定

**岗位名称**：产品设计师 / UX·UI 设计师（AI-Native Product Designer）

**一句话定位**：你是"体验的操盘手 + 一致性的守护者"。AI 负责生成布局、配色、组件、多方案，你负责判断好不好用、好不好看、统不统一。

**人做什么 vs AI 做什么**：

| 环节 | AI 负责（跑量、提速） | 人负责（判断、把关） |
|------|---------------------|---------------------|
| 信息架构 | 根据 PRD 生成页面清单、导航结构、跳转关系 | 判断架构是否符合用户心智、层级是否过深 |
| 交互流程 | 生成低保真线框图、交互原型、异常态设计 | 判断操作路径是否顺畅、有无多余步骤 |
| 视觉设计 | 生成多套配色/布局/组件方案、批量出图 | **定审美方向、选方案、保证品牌一致** |
| 设计系统 | 整理组件库、生成规范文档 | 定义规范标准、维护组件一致性 |
| 交付标注 | 自动标注间距/字号/切图、生成研发说明 | 检查标注准确性、对齐研发实现能力 |

**职责边界（红线）**：
- **审美方向和品牌一致性必须人定**。AI 能生成 100 个方案，但"哪个符合我们品牌调性"只有人能判断。
- AI 生成的界面常有"看起来很美但没法开发/不符合规范"的问题，交付前必须对齐研发实现能力和现有设计系统。
- **不要让 AI 每个页面各画各的**——会导致同一个产品里按钮长十种样子。一致性靠设计系统约束，不靠单页生成。

---

## 2. Skills（这个岗位必须具备的 AI 协作能力）

1. **PRD→IA 转化提示工程**：能引导 AI 从功能列表推导出合理的页面结构和导航层级，而不是简单罗列。
2. **设计意图描述能力**：会用精准的设计语言（布局、层级、留白、对比、品牌调性）向 AI 描述想要什么，而不是"帮我设计得好看点"。
3. **多方案评估与收敛**：让 AI 出多套方案，自己用体验和审美标准快速筛选、指出修改方向。
4. **设计系统思维**：懂得先建组件库和规范，再让 AI 在规范约束下生成页面，保证一致性。
5. **设计→研发交付能力**：能让 AI 生成研发看得懂的标注、切图说明，甚至直接产出可用的前端代码/样式。

## 3. AI 工具使用说明（最多 3 个，海外标杆 + 国内平替）

### 工具 ①：AI 界面设计 / 原型生成（文字或截图直接生成 UI）
- **海外标杆**：Figma（含 Figma AI / First Draft）、Uizard、Galileo AI —— 从文字描述直接生成可编辑的界面和原型。
- **国内平替**：**即时设计（含 AI 生成）/ MasterGo（摹客的设计工具）/ Pixso**。
  - 即时设计、MasterGo、Pixso 都是国产 Figma 平替，支持云端协作、组件库、AI 辅助生成，且访问稳定、中文友好；
  - 这三者是国内设计团队的主力工具，可直接承接团队协作和交付。
- **典型用法**：把 PRD 里某个页面的说明粘进去，让 AI 生成低保真线框图，再迭代到高保真。

### 工具 ②：AI 视觉/图像生成（配图、icon、插画、Banner、风格探索）
- **海外标杆**：Midjourney、DALL·E、Stable Diffusion —— 高质量视觉素材、风格化插画、概念探索。
- **国内平替**：**即梦（剪映/字节）、可灵（快手）、通义万相（阿里）、文心一格（百度）**。
  - 即梦、通义万相：中文语义理解好，出图快，适合生成产品配图、运营素材、图标风格探索；
  - 可灵：图生图、视频生成强，适合动效和宣传素材。
- **典型用法**：为空状态页生成插画、为功能生成图标风格草稿、探索视觉风格方向。

### 工具 ③：设计系统 / 组件规范管理
- **海外标杆**：Figma（Design System / Variables）、Zeplin（交付标注）。
- **国内平替**：**MasterGo / 即时设计的设计系统能力 + Mastergo/摹客的标注交付**。
  - 用国产工具自带的组件库、样式变量、自动标注功能，统一管理颜色/字号/间距/组件，并一键生成研发标注。

> **落地建议**：设计岗三件套 = **即时设计 或 MasterGo（界面+原型+设计系统+标注，一站式）+ 即梦/通义万相（视觉素材）+ 通用大模型（写文案、生成设计说明、辅助决策）**。国产一站式工具已能覆盖从原型到交付的全链路，无需强依赖 Figma。

## 4. 工作流程（可直接执行的 SOP）

> 核心链路：**PRD → 信息架构 → 交互流程/线框 → 视觉方案 → 设计系统约束 → 高保真定稿 → 标注交付**

### 第 1 步：从 PRD 推导信息架构（IA）
把 PRD 的功能详述喂给通用大模型，先理清"有哪些页面、怎么组织"。

**提示词模板**：
```
你是一名资深产品设计师。以下是产品岗交付的 PRD 功能详述。请你：
1. 梳理出这个产品/功能需要的全部页面清单；
2. 设计页面之间的导航结构和跳转关系（用层级列表或 Mermaid 流程图表示）；
3. 标注每个页面的核心内容和主要操作；
4. 指出信息层级是否过深（超过 3 层要预警），并给优化建议。
严格基于 PRD，不要新增 PRD 里没有的页面。

PRD 功能详述如下：
【粘贴 PRD 的功能详述 + 页面说明】
```

### 第 2 步：生成交互流程与低保真线框
明确了页面结构后，逐个页面生成线框图，重点覆盖异常态。

**提示词模板**（配合即时设计/MasterGo 的 AI 生成，或先用文字描述）：
```
针对【页面名】，请描述它的低保真线框布局：
1. 页面分区（顶部/内容区/底部，各放什么）；
2. 每个区域的元素（标题、列表、按钮、输入框等）及其位置关系；
3. 主要操作和点击后的跳转/反馈；
4. 必须包含的状态：加载中、空数据、加载失败、无权限、成功提示。
用结构化描述输出，方便我在设计工具里搭建线框。
```
> ⚠️ 空状态、失败态、加载态这些"非正常态"最容易被漏掉，一定要让 AI 显式列出并逐一设计——这是设计稿是否"可交付"的分水岭。

### 第 3 步：视觉风格探索（AI 出多方案，人选方向）
先定调性，再让 AI 批量出方案。

**提示词模板**（描述品牌调性）：
```
我们的产品是【一句话定位】，目标用户是【用户画像】，希望传达【关键词：如专业/年轻/科技/亲和】的感觉。
请给出 3 套视觉风格方向，每套包含：
- 主色 + 辅助色（给出色值）
- 字体风格建议
- 整体气质描述（举 1-2 个同类产品参照）
帮我判断哪套最匹配我们的定位。
```
> 选定方向后，用即梦/通义万相探索图标、插画、配图的具体风格；用 Midjourney 级工具做概念探索、国产工具做量产。

### 第 4 步：先建设计系统，再生成页面（保证一致性）
**这是 AI 设计最容易翻车的地方**。正确顺序是：**先定义组件和样式变量 → 再让 AI 在约束下生成页面**。

- 在即时设计/MasterGo 里建立：颜色变量、字号阶梯、间距规则、基础组件（按钮/输入框/卡片/导航）。
- 让 AI 生成页面时，明确要求"复用已有组件库，不要自创样式"。

**提示词模板**：
```
基于以下设计系统规范，为【页面名】生成高保真设计说明：
- 主色：【色值】辅助色：【色值】
- 字号：标题【x】正文【x】辅助【x】
- 间距基准：【x】px
- 已有组件：主按钮、次按钮、输入框、卡片、列表项（沿用，不要新造样式）
请说明页面上每个元素使用哪个组件、什么颜色、什么字号、什么间距。
```

### 第 5 步：高保真定稿（人把关审美和体验）
在设计工具里完成高保真，设计师逐页检查：
- **一致性**：同类元素样式是否统一（按钮、间距、圆角、图标风格）
- **可用性**：操作路径是否最短、关键操作是否突出、误操作是否可挽回
- **可实现性**：有没有研发难以实现的效果（复杂动效、非标准控件）

### 第 6 步：标注与切图交付（交给研发的关键）
用工具自动生成标注和切图，并附一份设计说明。

**生成设计交付说明的提示词**：
```
为以下页面生成一份给前端研发的设计交付说明，包含：
1. 页面用到的颜色/字号/间距（对应设计系统变量名）；
2. 每个交互元素的状态（默认/悬停/点击/禁用/加载）；
3. 动效说明（如有，描述触发条件、时长、缓动）；
4. 响应式/适配说明（不同屏幕尺寸如何变化）；
5. 需要研发注意的边界情况（文字过长、图片缺失如何处理）。
```

## 5. 交付物（本环节产出 → 交给"研发"环节）

一套 **《设计交付包 v1》**，包含：
1. **信息架构图**：页面清单 + 导航跳转关系
2. **高保真设计稿**：覆盖所有页面 + 所有状态（含空/错/加载态）
3. **交互说明**：操作路径、跳转逻辑、动效说明
4. **设计系统/规范**：颜色、字号、间距、组件库（研发按此实现，保证还原度）
5. **标注 + 切图 + 设计交付说明**：研发可直接对照实现的资源

> 这套《设计交付包 v1》是下一章"研发"环节的输入——工程师拿它来还原界面、实现交互。设计稿里的「组件库 + 标注 + 交互状态」直接决定前端要写哪些组件、每个组件有哪些状态。设计系统越规范，研发还原度越高、返工越少。

## 附：产品 → 设计 的咬合检查清单

在把设计交付包交给研发前，自查这几点，确保上下游对得上：

- [ ] PRD 里每个功能，都有对应的页面和交互设计（不漏做界面）
- [ ] PRD 里每个字段规则（必填、校验、错误提示），在设计里都有对应的展示和错误态
- [ ] PRD 里列出的每个异常处理，设计里都有对应的状态页（空/错/加载/无权限）
- [ ] 所有页面复用同一套设计系统，同类元素样式统一（无"十种按钮"）
- [ ] 标注和切图完整，研发不用猜颜色和间距
- [ ] 交付说明里标清了动效、响应式、边界情况，减少研发反问
