# 第九章：AI 工具全景图（选型速查）

> **推荐时点：2026 年 7 月。** AI 工具迭代极快，本章是一份"拿着就能选、就能买、就能用"的速查表。每个品类都给出**国外标杆 + 国内平替**，并标清**产品形态**（桌面客户端 / 终端 CLI / Web 云端 / 本地部署 / API / 浏览器插件 / 移动 App），方便你按团队的合规、网络、预算条件直接落地。
>
> ⚠️ **时效声明**：本章信息截至 2026 年 7 月。AI 领域改名、停服、被收购是常态（例如 Codeium 已改名 Windsurf 且公司主体被拆解）。**建议每季度校准一次**——用第八章的"标准工具栈"季度评审机制刷新本表。看到本章时若已过半年以上，请重新核实各工具现状。

---

## 0. 怎么读这张表

**形态缩写对照**：

| 缩写 | 含义 | 对团队意味着什么 |
|------|------|-----------------|
| Web | 网页/云端使用 | 打开浏览器即用，无需安装，数据在云端 |
| 桌面 | 桌面客户端（Win/Mac） | 本地安装的 App，体验更稳 |
| CLI | 命令行/终端工具 | 工程师在终端里用，适合自动化和脚本 |
| 插件 | IDE/浏览器插件 | 嵌入现有工具（VS Code、浏览器）里用 |
| App | 移动端 App | 手机/平板上用 |
| API | 开放接口 | 供自研系统调用，能力嵌进自己产品 |
| 本地部署 | 可私有化/本地运行 | **数据不出公司**，敏感场景首选 |

**选型三条快速判断**（呼应第八章）：
1. **数据敏感 → 优先"本地部署"形态**（数据不出公司）。
2. **国内团队日常用 → 优先国内工具**（访问稳、中文好、数据在境内、合规风险低）。
3. **要嵌进自研产品 → 看有没有 API**。

| **豆包** | 国内(字节) | Web/桌面/App/API | 响应快、生态全、国内日常首选之一 |

## 8. 落地建议：怎么用这张表做选型

1. **先定通用大模型主力**（第 1 节）——全公司统一 1-2 个，这是所有工作的底座。
2. **按岗位配三件套**（第 7 节）——照着采购清单配，避免各买各的。
3. **敏感数据场景单独走本地部署**（第 6 节）——数据不出公司。
4. **统一企业账号采购**（呼应第八章成本管理）——企业版比个人散买划算、好管理。
5. **季度校准本表**——AI 工具变化快，用第八章的季度评审机制刷新，淘汰停服/低效的、引入更优的。

> **一句话总结**：这张表解决"用什么"，前八章解决"怎么用"。两者配合，团队既能快速选型采购，又能规范高效地把工具用起来。

## 附：本章时效核对清单（下次校准时逐项检查）

- [ ] 通用大模型的主力是否仍是当前最优（模型迭代快，半年可能换代）
- [ ] 有无工具改名/停服/被收购（如已发生的 Codeium→Windsurf）
- [ ] 有无值得纳入的新工具（尤其国产新品）
- [ ] 各工具的形态是否有变化（如新增本地部署/CLI/API）
- [ ] 合规状况是否变化（数据出境政策、备案要求等）
- [ ] 价格/计费模式是否变化（影响成本测算）
