# 第六章：媒体推广（Marketing / Growth）

> 单产品流水线的最后一环。发布岗交来一份稳定上线的产品 + Release Note + 上线数据基线，推广岗要把它变成用户看得见、愿意传播、能带来增长的声量。AI 原生模式下，市场/增长人员不再是"一篇篇手写文案、一个个平台搬运的人"，而是"用 AI 把一个卖点批量改写成多平台内容、规模化生产素材、用数据驱动投放优化，自己专注在策略、调性和真实性把关"的人。

---

## 0. 本章要解决的核心问题

**如何把一个上线的产品，变成多平台的传播声量和增长？**

难点不是"写不出文案"，而是**中国市场平台极度碎片化**——公众号、小红书、抖音、B站、知乎、视频号、微博，每个平台的用户、调性、内容形态都不同。同一个卖点，在小红书要写成种草笔记，在知乎要写成专业解答，在抖音要变成 15 秒短视频脚本。

AI 的价值正在于此：**一次提炼核心卖点，批量适配 N 个平台**，把"人肉搬运改写"的重复劳动接管，让推广人员专注在三件事——**定策略**（打哪个人群、哪个平台）、**保调性**（符合品牌、符合平台语境）、**守真实**（不虚假宣传、不编造数据）。

---

## 1. 角色设定

**岗位名称**：市场 / 增长 / 内容运营（AI-Native Growth Marketer）

**一句话定位**：你是"传播的策划人 + 真实性的守门人"。AI 负责批量产内容、铺多平台、跑数据，你负责定打法、把调性、守住不夸大不造假的底线。

**人做什么 vs AI 做什么**：

| 环节 | AI 负责（跑量、提速） | 人负责（判断、把关） |
|------|---------------------|---------------------|
| 卖点提炼 | 从 Release Note 提炼卖点、生成传播角度 | **定核心信息、定主打人群** |
| 内容生产 | 批量生成多平台文案、脚本、标题 | 审调性、审真实性、选最优 |
| 素材制作 | 生成配图、短视频、海报、封面 | 把关品牌一致性和审美 |
| 多平台适配 | 把一个卖点改写成各平台形态 | 判断是否符合平台语境和规则 |
| 投放优化 | 分析投放数据、建议优化方向 | **定预算、定投放策略、拍板加减投** |
| 效果复盘 | 汇总数据、生成复盘报告 | 判断真实效果、定下一步 |

**职责边界（红线）**：
- **真实性是绝对红线**。AI 会为了"效果好"编造用户评价、夸大功能、虚构数据——所有对外内容必须人工核实，不虚假宣传（这也是法律红线，《广告法》《反不正当竞争法》）。
- **品牌调性和核心信息必须人定**。AI 能写 100 个版本，但"我们要传达什么、不碰什么"只有人能把关。
- **投放预算决策必须人拍板**。AI 给优化建议，加投减投由人定。
- 涉及**敏感词、平台规则、合规**的内容，发布前必须人工审核（尤其医疗、金融、教育等强监管行业）。

---

## 2. Skills（这个岗位必须具备的 AI 协作能力）

1. **卖点提炼与传播策划**：能引导 AI 从产品功能里提炼出"用户在意的价值"，而不是罗列功能参数。
2. **多平台内容改写提示工程**：会针对不同平台（小红书/知乎/抖音/公众号）的语境和调性，让 AI 精准适配，而不是一稿多发。
3. **AI 素材生成**：会用文生图/文生视频工具批量产出配图、短视频、封面、海报，并保持品牌一致。
4. **数据驱动优化**：能把投放和传播数据喂给 AI 做分析，识别哪些内容/渠道有效，指导预算分配。
5. **真实性与合规审查**：始终对 AI 生成内容做事实核查和合规过滤，守住不夸大、不造假、不违规的底线。

---

## 3. AI 工具使用说明（最多 3 个，海外标杆 + 国内平替）

### 工具 ①：通用大模型（卖点提炼、多平台文案、复盘）
- **海外标杆**：ChatGPT / Claude —— 内容创意、多风格改写、长文和脚本生成能力最强。
- **国内平替**：**豆包、DeepSeek、Kimi、通义千问 + 腾讯混元**。
  - 豆包：文案创意、多平台改写快，生态贴近国内内容语境；
  - DeepSeek：策略分析、数据复盘逻辑强；
  - Kimi/通义：长文本，适合一次生成系列内容。
- **典型用法**：把 Release Note 喂进去，让它提炼 3 个核心卖点，再分别改写成小红书笔记、知乎回答、抖音脚本、公众号推文。

### 工具 ②：AI 图像 / 视频 / 素材生成（规模化产素材）
- **海外标杆**：Midjourney、Runway、Pika、CapCut —— 高质量配图、AI 视频、动效。
- **国内平替**：**即梦（字节）、可灵（快手）、通义万相（阿里）、剪映（含 AI 功能）、腾讯智影、稿定设计**。
  - 即梦 / 通义万相：产品配图、海报、封面、宣传图；
  - 可灵 / 剪映：AI 短视频、口播视频、混剪；
  - 稿定设计：模板化快速出多平台尺寸的营销物料。
- **典型用法**：为一个卖点批量生成小红书封面、抖音视频、朋友圈海报，尺寸和风格一键适配。

### 工具 ③：投放 / 数据分析平台（增长优化）
- **海外标杆**：Google Ads、Meta Ads（含 AI 优化）、GA4 —— 智能投放 + 效果分析。
- **国内平替**：**巨量引擎（抖音/今日头条）、腾讯广告（微信/朋友圈/视频号）、小红书聚光、百度营销 + 巨量云图/数据银行做分析**。
  - 巨量引擎、腾讯广告：国内两大主力投放平台，自带 AI 智能投放和优化；
  - 小红书聚光：种草和搜索场景投放；
  - 结合国产大模型分析后台数据，找高效内容和人群。
- **典型用法**：AI 分析各渠道 ROI、各内容互动率，建议预算往高效渠道倾斜。

> **落地建议**：推广岗三件套 = **豆包/DeepSeek（文案+策略+复盘）+ 即梦/可灵/剪映（素材生产）+ 巨量引擎/腾讯广告（投放优化）**。核心是「大模型产内容 + AI 工具产素材 + 投放平台做增长」。

## 4. 工作流程（可直接执行的 SOP）

> 核心链路：**卖点提炼 → 定策略 → 多平台内容生产 → 素材制作 → 分发投放 → 数据复盘 → 迭代**

### 第 1 步：从 Release Note 提炼核心卖点（人定主信息）
拿发布岗的 Release Note 和上线数据，先提炼卖点。

**提示词模板**：
```
你是一名资深增长营销专家。以下是产品的 Release Note（本次上线的功能/改进/修复）。请：
1. 提炼 3 个最能打动用户的核心卖点（讲用户价值，不是功能参数）；
2. 每个卖点标注：针对什么人群、解决什么痛点、一句话钩子；
3. 指出哪个卖点最适合作为主打。
严格基于 Release Note，不要夸大或编造产品没有的能力。

Release Note：【粘贴】
```
> ⚠️ **主打卖点和目标人群由人拍板**——这决定后面所有内容的方向。

### 第 2 步：定传播策略（人主导）
基于卖点和人群，定"打哪些平台、什么节奏、什么预算量级"。让 AI 给建议，人决策。

**提示词模板**：
```
针对以下卖点和目标人群，建议传播策略：
1. 优先主攻哪些平台（小红书/抖音/知乎/公众号/B站/视频号），为什么；
2. 各平台的内容形态和调性建议；
3. 节奏建议（预热/上线/持续）；
4. 免费内容(自然流)和付费投放的配比建议。
卖点：【】目标人群：【】预算量级：【】
```

### 第 3 步：多平台内容批量生产（AI 跑量，人选优）
一个卖点，批量适配多平台。**这是 AI 最提效的一步。**

**多平台改写提示词模板**：
```
把以下卖点分别改写成各平台的内容，注意各平台调性：
- 小红书：种草笔记，第一人称、真实体验感、emoji、痛点+解决、5-8个话题标签；
- 知乎：专业解答，理性、有逻辑、举例、克制不硬广；
- 抖音/视频号：15-30秒短视频脚本，前3秒钩子、口语化、有记忆点、结尾引导；
- 公众号：图文推文，标题党但不虚假、结构清晰、有小标题；
- 微博：话题短文案，简短、有梗、带话题#。
每个平台给2个版本供我选。不要编造用户评价和数据，功能描述必须与实际一致。

卖点：【】
```
> AI 出多版本，人选最符合调性、最真实的那个。**真实性核查在这一步做——凡是"用户都说""效果提升300%"这类，没证据的一律删。**

### 第 4 步：素材制作（AI 产图产视频，人把关品牌）
用即梦/可灵/剪映/稿定，为选定内容批量产素材。

**素材生成提示词模板（配文生图工具）**：
```
为【平台+内容主题】生成配图/封面：
- 风格：符合我们品牌调性【描述，或参考设计岗的视觉规范】；
- 主色：【品牌色】；
- 尺寸：【小红书3:4 / 抖音9:16 / 公众号首图2.35:1】；
- 画面要素：【描述】；
- 不要出现：【避免的元素、错别字、误导性内容】。
```
> 品牌一致性要对齐第三章设计岗的视觉规范，别让推广素材和产品本身"两张脸"。

### 第 5 步：分发与投放
- **自然流**：按平台节奏发布内容，用矩阵账号铺量。
- **付费投放**：在巨量引擎/腾讯广告/小红书聚光设置定向，用平台 AI 智能投放起量，小预算测试跑出高效素材再放量。

### 第 6 步：数据复盘与迭代（AI 分析，人定方向）
把各平台互动数据、投放数据喂给 AI 分析。

**复盘提示词模板**：
```
以下是本轮传播的数据（各平台曝光/互动/转化，各投放渠道花费/ROI/获客成本）。请：
1. 找出表现最好和最差的内容和渠道，分析原因；
2. 对比上线数据基线，评估推广对产品指标的实际拉动；
3. 给出下一轮优化建议：内容方向、预算分配、渠道取舍。
数据只做客观分析，不要为了好看而美化结论。

数据：【粘贴】上线数据基线：【粘贴发布岗提供的基线】
```
> 用发布岗提供的**上线数据基线**做对比，才能分清"是推广带来的增长"还是"产品本身的自然量"。

---

## 5. 交付物（本环节产出：单产品流水线的终点）

**一轮完整的传播落地 + 效果沉淀**，包含：
1. **多平台内容矩阵**：已发布的各平台内容（真实、合规、符合调性）
2. **传播数据报告**：曝光/互动/转化/ROI，对比上线基线的增长评估
3. **高效素材/内容库**：跑出来的爆款内容和素材，可复用到下一轮
4. **优化建议**：下一轮传播的方向、渠道、预算分配
5. **反馈回流**：用户在传播中暴露的新需求/吐槽 → **回流给需求岗（第一章），进入下一轮迭代**

> **闭环点**：推广不是流水线的终点，而是**下一轮的起点**。传播中收集的用户反馈、市场反应，回流到需求岗，整条流水线重新转起来。这就是 AI 原生公司的增长飞轮。

---

## 附：发布 → 推广 的咬合检查清单

在大规模推广前自查：

- [ ] 产品已稳定上线、监控正常（没上稳不大推，别把 bug 放大给全网）
- [ ] 卖点都基于真实 Release Note，无夸大、无编造功能
- [ ] 所有对外内容经过真实性核查（无虚假用户评价、无编造数据）
- [ ] 内容符合各平台规则和相关法规（《广告法》等），敏感行业已合规审核
- [ ] 推广素材品牌调性与产品/设计一致（无"两张脸"）
- [ ] 有上线数据基线可对比，能分清推广增量和自然量
- [ ] 传播中的用户反馈有回收机制，能回流给需求岗
